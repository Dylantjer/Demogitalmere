HTML

< html > is de bron van de pagina.

< !doctype html > zorgt er voor om aan te geven welke HTML versie er word gebruikt.

de "lang" attribuut bepaald het taal van het HTML. Dit verklaart of stelt het taal van HTML vast zodat
dit makkelijker is voor de search engines om jouw website te kunnen vinden

De < meta > element is er om het character set, descriptie voor de pagina, sleutelworden, auteur van de document en vieuwport instelling in te stellen.
Dit is handig voor search engines of hoe de pagina content vertoont moet worden en ook instelling voor het refreshen van de pagina (Zoals om hoeveel seconden er refreshed moet worden.)
De < Viewport >element bepaalt hoe je pagina vertoont word afhankelijk van de apparaat. De "width=device-width" zorgt daarvoor. De "initial-scale=1.0" zorgt ervoor hoeveel er ingezoomed is op jouw pagina. < base >zorgt voor dat er een standaard URL op je pagina.

Links

de < a > tag wordt gebruikt voor links.

Binnen het linken kun je met een attribuut er voor zorgen hoe de andere website wordt geopend:

  • self, opent de de website in hetzelfde tab.
  • blank, opent de document in een nieuwe window of tab.
  • parent, opent de document in de parent frame
  • _top, opent de document in de volle body van de window.
  • Iframe, opent de link binnen de iFrame element.

Je hebt absolute URLs en Relative URLS.
Absolute URLS zijn links naar website's. Relative URLS zijn links binnen jouw lokale bestanden. denk aan waar je plaatjes in je PC staat enz.

Style / CSS

Met de "style" attribuut kun je CSS of vormgeving toepassen op je HTML.
Er zijn verschillende manieren om "style" toe te passen:

  • Inline
  • Internal
  • External

Inline

Inline word gebruikt om 1 element aan te passen met vormgeving.

Internal

Internal style is vastgesteld in de "Head" element. Dit word gebruikt om een bepaalde selectie te vormgeven. Hiermee kan je bijvoorbeeld alle

< h1 > elementen binnen de HTML broncode een andere font hebben, of een andere kleur of stijl.

External

Dit is een externe CSS bestand die gekoppeld aan je HTML broncode binnen je head element.

HTML Methods

De "Method" attribuut zorgt dat je de "Post" en "Get" elementen kunt gebruiken voor het verzenden van gegevens met behulp van de attribuut "action".

de POST element voegt de form data in het lichaam van de HTTP request (Dus niet in de adresbalk). Dit kan worden gebruikt voor grote data. Deze kunnen niet gebookmarked worden.
de Formenctype attribuut zorgt er voor hoe de formuliergegevens moeten worden ingevoerd. (DIT WORDT GEBRUIKT VOOR FORMS MET DE METHODE ATTRIBUUT =POST

Object en Embed element

Object element word vaak gebruikt om plug-ins in werking te zetten, denk aan andere HTML bestanden, of PDF readers, Flash player enz.
Het verschil tussen embed en Object element is:

Object:Dit is een methode om een bestand toe te voegen aan je website die je bij de bron (het lokale bestand zelf) kan aanpassen.
Embed: Dit is een methode om bestanden toe te voegen zonder dat er aanpassingen in kunnen komen.

Web storage & Web Workers API

Web storage is één van de meerdere API's waar de data van jouw website opgeslagen kan worden in diegene zijn lokale bestanden. Dit is handig voor bijvoorbeeld het onthouden van inlog-gegevens.
Dit is tenslotte ook handig voor de performance van de website, de info wordt nooit gestuurd naar de server. De hoeveelheid dat opgeslagen kan worden is groter dan 5 MB.

Er zijn 2 manieren om data op te slaan:
Via Local storage: "window.localStorage" dit slaat data op de bezoekers computer, zonder dat het over een bepaald tijd weg is.
Via Session Storage: "window.sessionStorage" Dit slaat data op voor één sessie, zodra de pagina gesloten is zijn deze gegevens kwijt.

Een webworker API is een Javascript dat in de achtergrond aan het draaien is zonder dat dit invloed heeft op de performance van de pagina.
